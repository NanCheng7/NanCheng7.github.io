二、Git

1、工作原理

Workspace：工作区
Index / Stage：暂存区
Repository：仓库区（或本地仓库）
Remote：远程仓库

2、命令行

代码 作用
git config –global user.name “xxx” 添加用户名
git config –global user.email “xxx” 添加邮箱
git config –list 查看配置
pwd 命令用于当前的目录
git init 把当前目录变成git可以管理的仓库
git add 把文件提交到暂存区
git commit -m “message” 将文件提交，message是提交的注释
git status 查看是否还有文件没有提交
git diff 查看文件到底修改了什么额呢绒
git log 查看历史记录
git log –pretty=oneline 查看简介的历史记录
git reset –hard HEAD^ 回退到上一个版本，如果需要回退两个版本，则git reset –hard HEAD^^
git reset –hard HEAD~10 回退到前十个版本
cat 查看文件内容
git reset –hard <版本号> 回退到对应版本号
git reflog 查看版本号

3、理解工作区与暂存区的区别

第一步：是使用 git add 把文件添加进去，实际上就是把文件添加到暂存区。

第二步：使用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支上。

4、Git撤销修改和删除文件操作

1、撤销修改:

  • 直接手动更改去掉那些需要的文件，然后add添加到暂存区，最后commit掉。

  • 使用 git reset –hard HEAD^

  • 使用git checkout – <file> 可以丢弃工作区的修改.

    命令 git checkout –readme.txt 意思就是，把readme.txt文件在工作区做的修改全部撤销，这里有2种情况，如下：

    1.readme.txt自动修改后，还没有放到暂存区，使用 撤销修改就回到和版本库一模一样的状态。
    2.另外一种是readme.txt已经放入暂存区了，接着又作了修改，撤销修改就回到添加暂存区后的状态.

2、删除文件

​ rm <file> 删除文件

如果我想彻底从版本库中删掉了此文件的话，可以再执行commit命令 提交掉

二、建立云端仓库

​ 提交文件，注释massege

​ 建立连接、
